## ADR-013: LaunchAgent over manual process management
**Date:** 2026-06-13
**Decision:** Bot runs as macOS LaunchAgent via launchctl, not as a manually started terminal process.
**Why:** Manual process dies when terminal closes or Mac sleeps. Scheduler never fires. LaunchAgent provides KeepAlive restart, runs at login, logs to store/bot.log and store/bot.error.log.
**Consequences:** Code changes require launchctl unload/load cycle to take effect — not just Ctrl+C and restart.
**Revisit when:** Moving to a server or always-on machine.

## ADR-012: Company logo on cover slide
**Date:** 2026-06-12
**Decision:** Local PNG files (assets/logos/) as primary source,
Google S2 favicon API as long-tail fallback
**Why:** Clearbit Logo API DNS is dead. Google S2 returns favicons
with background fill (no transparency) — acceptable for unknown
companies, unacceptable for primary brands. Local PNGs provide
clean transparent logos for the 10-15 companies that appear most.
**Consequences:** Manual curation of assets/logos/ required for
priority companies. S2 fallback shows white box on dark slides
for companies not in local set — known limitation, not a bug.
**Revisit when:** A reliable transparent-logo API emerges.

## ADR-011: Pexels as image fallback between OG and FLUX
**Date:** 2026-06-12
**Decision:** OG image → Pexels search → FLUX generation → None
**Why:** FLUX HF keys were empty since project start — silently
non-functional. Pexels is free, fast, high quality, no generation
wait. Portrait orientation filter gives carousel-ready images.
**Consequences:** Pexels requires PEXELS_API_KEY in .env.
FLUX remains as last resort if Pexels fails.
**Revisit when:** FLUX keys are confirmed working.

## ADR-010: Self-learning scrape reliability via feedback.py
**Date:** 2026-06-12
**Decision:** Track scrape success/failure per domain in
store/feedback_log.json. Apply reliability multiplier in scorer.py
after 3+ attempts. Auto-flag domains below 30% reliability.
**Why:** Manual blocklist (sources.yaml) requires human maintenance.
Reliability scoring makes the blocklist self-managing over time.
**Consequences:** Needs 30+ days of data before multiplier has
real effect. Domains with < 3 attempts are unaffected.
**Revisit when:** 30 days of operational data accumulated.

## ADR-009: Source fallback on 403 — search alternative coverage
**Date:** 2026-06-12
**Decision:** On scrape failure, search Serper for alternative
coverage of the same story on TC/Verge/Wired/Ars before failing.
**Why:** A good story shouldn't be lost because the source is
paywalled. Alternative coverage of the same event is often
available on open sources.
**Consequences:** Adds one Serper API call per failed scrape.
Alternative URL is logged to run_log.json for traceability.
**Revisit when:** Serper quota becomes a concern.

## ADR-008: Jina Reader API as scraper fallback for JS-rendered pages
**Date:** 2026-06-12
**Decision:** 7-strategy scraper: p tags → article → content divs
→ main → largest div → mobile UA → Jina Reader API (r.jina.ai)
**Why:** JS-rendered pages (React/Remix SPAs like Pandaily) return
0 chars from all HTML strategies. Jina renders and returns clean
text. No headless browser dependency required.
**Consequences:** Jina is a third-party service with unknown SLA.
Hard paywalls (inc.com) fail even Jina — these go to blocklist.
**Revisit when:** Jina reliability becomes an issue.

## ADR-007: Signal and quote slides fill 3/4 of canvas
**Date:** 2026-06-12
**Decision:** Signal headline at 126px, quote text at 96px —
content fills ~75% of 1350px canvas height.
**Why:** At smaller sizes (78px/63px), slides had large empty
zones below content. Users would exit by slide 2. Each slide
must justify its existence as a swipe.
**Consequences:** Long headlines or quotes may overflow — writer
prompt enforces word limits to prevent this.
**Revisit when:** Body image scraping adds visual density to
compensate for shorter text blocks.

## ADR-006: Stat slide whitespace — deferred
**Date:** 2026-06-12
**Decision:** Stat slide known to have excess whitespace on
short content. Deferred — not fixed.
**Why:** The number (260px) anchored to top leaves empty space
below on short context text. Fix requires either: minimum content
requirements before stat slide is selected, or a visual element
(body image) to fill the space. Body image scraping is parked.
**Consequences:** Stat slides look sparse on short articles.
Model may over-select stat slides for stories that don't warrant them.
**Revisit when:** Body image scraping is implemented (PARKING_LOT).

## ADR-005: Caption generated by LLM as top-level JSON field
**Date:** 2026-06-11
**Decision:** caption is a top-level field in writer output,
not derived from CTA slide headline.
**Why:** CTA headlines are follow hooks ("Stay Ahead of the AR
Revolution") — wrong content for an Instagram caption. LLM
generates a factual 3-4 sentence summary as a separate field.
**Consequences:** Caption quality depends on LLM following the
50-word minimum instruction. Baked into both WRITER_PROMPT and
_SCHEMA_ENFORCEMENT to prevent regression.
**Revisit when:** Never — this is the correct architecture.

## ADR-004: Schema enforcement injected at call time in writer.py
**Date:** 2026-06-11
**Decision:** Field names injected via _SCHEMA_ENFORCEMENT block
in writer.py at call time. design_constants.py WRITER_SCHEMA is
the canonical schema but not directly injected.
**Why:** llama-3.3-70b-versatile ignores abstract schema
descriptions. Explicit field name enforcement ("headline not
title, attribution not author") in the prompt is required.
**Consequences:** Two sources of truth (WRITER_SCHEMA and
_SCHEMA_ENFORCEMENT) must stay in sync manually.
**Revisit when:** Model improves at schema following.

## ADR-003: Groq over Gemini
**Date:** 2026-06-11
**Decision:** Groq (llama-3.3-70b-versatile) as primary LLM.
llama-3.1-70b-versatile as fallback on 429.
**Why:** Gemini free tier quota exhausted during interactive
testing. Org policy blocked billing on GCP project. AI Studio
workaround also hit limits. Groq free tier sufficient for
current volume. 8B model (llama-3.1-8b-instant) failed schema
compliance — 70B required.
**Consequences:** No multimodal capability. Groq rate limits
apply at scale. Schema enforcement prompt required.
**Revisit when:** Gemini quota available or volume increases
beyond Groq free tier.

## ADR-002: Flat files over database
**Date:** 2026-06-11
**Decision:** store/*.json over ChromaDB, SQLite, or any DB.
**Why:** v1 used ChromaDB — added complexity without value at
this scope. Flat files are diffable, debuggable, portable, and
readable by any agent without a driver.
**Consequences:** No query capability. Manual cleanup needed as
files grow. No transactions — last-write-wins on concurrent writes
(acceptable: single-process system).
**Revisit when:** run_log.json exceeds 10MB or query needs arise.

## ADR-001: Instruction-driven slide selection over hardcoded anatomy
**Date:** 2026-06-11
**Decision:** Gemini/Groq selects slide types from a palette
based on story content. No fixed slide order beyond cover + cta.
**Why:** v1 had rigid slide anatomy that produced thin slides on
short stories and couldn't adapt to story type. Instruction-driven
selection lets the model choose 2-6 slides appropriate to the
content.
**Consequences:** Model judgment determines slide quality.
Ground-or-abstain must be enforced in prompt. Slide count varies
per story (4-8 total including cover + cta).
**Revisit when:** Never — this is the correct architecture.
