Frameshift v2 — Writer Contract

Last updated: 2026-06-11  ·  @ai.xr.frameshift  ·  writer.py + design_constants.py

Overview

The writer takes an article URL and produces a structured JSON carousel. This document is the contract — Groq must output exactly this structure or validation fails and the pipeline raises a ValueError.

The contract lives in two places:

  • watcher/design_constants.py — WRITER_SCHEMA and WRITER_PROMPT
  • watcher/writer.py — _SCHEMA_ENFORCEMENT block injected at call time; validate() function

When the schema or prompt changes, update design_constants.py first, then writer.py, then this document. Run tests/unit/test_writer_schema.py to confirm validation still catches bad output.

Output Schema

The full top-level structure Groq must return. The caption field is a top-level sibling of slides, not a slide type. 

{
  "caption": "3-4 sentences summarising the story. Min 50 words. Third person,
              factual, no hype. Lead with the most interesting fact. Do NOT
              mention The Frameshift or @ai.xr.frameshift.",

  "slides": [
    {
      "type": "cover",
      "headline": "...",
      "category": "...",
      "image_query": "..."
    },

    // 2–6 middle slides — any combination from the palette
    // (stat, quote, context, contrast, timeline, explainer, signal)
    // Max 2 of any single type. Ground or abstain: no invented facts.

    {
      "type": "cta",
      "headline": "..."
    }
  ]
}

Slide Schemas

Field names must match exactly — the validator checks keys, not just values. All word limits are hard limits. Required fields must be present and non-empty. Optional fields may be omitted or false.

Cover always first

FieldTypeLimitNotes
headline req string max 12 words Full-slide headline; renderer dynamic-sizes 96→54px to fit
category req string max 4 words Shown as pill in uppercase; min 2 chars
image_query req string 5–8 words Used for OG image search or FLUX generation prompt

Stat one number that reframes the story

FieldTypeLimitNotes
eyebrow req string max 4 words Label above the number (e.g. "Market Signal")
number req string — e.g. "$14B" or "73%"; min 1 char; rendered at 260px
unit req string max 8 words Describes what the number measures; min 1 char
context req string max 35 words One or two sentences adding meaning to the number

Quote direct quote > paraphrase

FieldTypeLimitNotes
text req string max 40 words The direct quote; must be verbatim from the article
attribution req string — Format: Name, Title — Organisation

Context what happened + optional image

FieldTypeLimitNotes
kicker req string max 3 words Short label above headline (e.g. "What happened")
headline req string max 15 words What happened — rendered at 86px EB Garamond
body req string max 40 words Supporting detail below headline or image
use_image opt boolean — Set true only when article image adds genuine value. Renderer shows image zone only if both use_image=true and image_obj is available — never a grey placeholder.

Contrast two competing positions

FieldTypeLimitNotes
kicker req string max 3 words Short label (e.g. "The divide")
label_a req string max 3 words Dark cell label (e.g. "Old model")
value_a req string max 20 words Dark cell value — displayed in WHITE EB Garamond 54px
label_b req string max 3 words Light cell label
value_b req string max 20 words Light cell value — displayed in NEAR_BLACK EB Garamond 54px
takeaway req string max 25 words Single sentence below both cells synthesising the tension

Timeline sequence and momentum

FieldTypeLimitNotes
kicker req string max 3 words Short label (e.g. "How we got here")
events req array — Ordered list of event objects; typically 3–5 items
events[].date req string — Year or date string (e.g. "2023", "Q4 2024")
events[].event req string max 12 words What happened at this point in the sequence

Explainer reader needs a concept first

FieldTypeLimitNotes
kicker req string max 3 words Short label (e.g. "Worth knowing")
question req string max 15 words The concept as a question — rendered at 86px EB Garamond
answer req string max 55 words Plain-language answer. Any **bold** markers are stripped before rendering.

Signal one forward-looking implication

FieldTypeLimitNotes
kicker req string max 3 words Short label (e.g. "What this means")
headline req string max 15 words Forward-looking implication as a headline; rendered at 126px — the largest headline in the palette
implication req string max 45 words Expanded explanation of the implication
tag req string max 4 words Pill label shown below implication (e.g. "Watch this space"); min 1 char

CTA always last

FieldTypeLimitNotes
headline req string max 20 words Story-specific follow hook — not generic ("Follow for more"). Must give a reason to follow tied to this specific story. Min 10 chars.

Slide Palette Rules

Rules Groq must follow when selecting and ordering middle slides:

  • cover and cta are always present — fixed first and last, respectively
  • Choose 2–6 middle slides from: stat, quote, context, contrast, timeline, explainer, signal
  • Max 2 of any single slide type in one carousel
  • Ground or abstain: every claim must come from the article text. If no stat exists, omit the stat slide. See Ground or Abstain.
  • stat: use when there is a single number that reframes the story
  • quote: use when a direct quote carries more weight than paraphrase
  • context: use when the reader needs to know what happened before anything else lands
  • contrast: use when two competing things are the core tension of the story
  • timeline: use when sequence and momentum matter — the order of events is the point
  • explainer: use when the reader needs a concept explained before the implication makes sense
  • signal: use for one forward-looking implication — what this means in the next 12–36 months
  • If the story is thin, 2 middle slides is correct — do not pad to fill the maximum
  • Order slides in narrative order: context → what it means → what to watch, not arbitrary

Validation Rules

Applied in writer.py:validate() after each Groq response. A ValueError is raised on any failure — the pipeline does not silently accept invalid output.

RuleDetail
First slide must be cover slides[0]["type"] == "cover"
Last slide must be cta slides[-1]["type"] == "cta"
2–6 middle slides Total slides count: 4–8 (cover + 2–6 middle + cta)
All types must be known Every type value must exist in renderer.RENDERERS.keys()
All field names must match schema Checked via _SCHEMA_ENFORCEMENT block injected into prompt; validator checks required keys present
Minimum field lengths _FIELD_MIN dict: category ≥ 2 chars; number, tag, unit ≥ 1 char
caption minimum length ≥ 80 characters
cta.headline minimum length ≥ 10 characters

Ground or Abstain

Every claim in every slide must be grounded in the article text. If the article does not contain the information needed for a slide type, that slide type must be omitted — not invented, not approximated. This rule is wired into WRITER_PROMPT in design_constants.py, not just documented here.

"Every claim must come from the article — ground or abstain."
— from WRITER_PROMPT, RULES section
Correct
Article has no statistics. Writer selects 2 middle slides (context + signal) and omits the stat slide entirely.
Incorrect
Article mentions vague growth. Writer invents "$3B market by 2027" to fill a stat slide. This violates ground-or-abstain regardless of how plausible the number sounds.
Correct
Article has a direct quote. Writer uses it in a quote slide verbatim. Attribution matches the name and title given in the article.
Incorrect
Article paraphrases a source. Writer constructs a quote in first person that the source never said. This is fabrication — omit the quote slide and use context instead.

The rule applies to all fields: stat numbers, quote text, timeline dates, and signal implications must all trace back to the article. Context can be inferred from the article's framing, but must not introduce claims the article doesn't support.

Caption Requirements

The caption field is a top-level sibling of slides — not a slide type. It becomes the Instagram post caption, attached to the first slide of the media group. Format in Telegram delivery: {caption}\n\n{source_url}.

RequirementDetail
LengthExactly 3–4 sentences; minimum 50 words; minimum 80 characters
Lead sentenceMust open with the single most interesting or surprising fact from the article
Sentence 2–3Explain why the fact matters — context or implication
VoiceThird person, factual, no hype. No exclamation marks.
Do NOTRestate the carousel headline verbatim
Do NOTMention "The Frameshift" or "@ai.xr.frameshift"
Do NOTInclude hashtags — these are added separately if needed
Field nameMust be exactly "caption" — validated by name
Example — good caption
Asus and Xreal have shipped the first AR glasses built exclusively for gaming, running at 120Hz without a tethered phone. The device marks the first time a major PC brand has co-designed AR hardware from the ground up, signalling a shift from general-purpose to use-case-first AR. Every major display category — 4K, high-refresh monitors — found its mass market through gaming first; AR appears to be following the same path.
