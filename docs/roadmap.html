Phase Detail

Phase 0 — Foundation Shipped

0A — Design System

Full slide palette designed as React artifact, approved by Snow. Cover template matched to existing brand sample. 8 slide types defined: cover (fixed), stat, quote, context, contrast, timeline, explainer, signal, cta (fixed). Lavender #b8a9d9 accent, warm off-white #f7f4ef content slides, pure black #000000 CTA.

Key decisions:
  • Slide structure is instruction-driven, not hardcoded — LLM selects from palette based on story, 2–6 middle slides, ground-or-abstain enforced
  • Cover headline dynamically sized 96px→54px to fill bottom third of slide
  • Stat slide content top-anchored, accent bar pinned to bottom
  • Images only on cover (full-bleed) and context slides (optional zone)

0B — renderer.py

Built: watcher/renderer.py — renders all 8 slide types to 1080×1350 PNG.

Key decisions:
  • Font fallback: checks fonts/ for EB Garamond + DM Sans, falls back to DejaVu if missing — pipeline never dies on missing fonts
  • Slide dispatch via RENDERERS dict — adding a new slide type is one function + one dict entry
  • render_carousel(slides_json) is the single public interface

0C — design_constants.py

Built: watcher/design_constants.py — single source of truth for all design values, writer schema, and prompt template.

0D — writer.py

Built: watcher/writer.py — scrape + Groq → validated slide JSON. LLM switched from Gemini (quota exhausted) to Groq llama-3.3-70b-versatile.

0E — Real fonts

EB Garamond (Regular, Italic, Bold) + DM Sans (Regular, Medium, Bold) installed at ~/frameshift_v2/fonts/.

Phase 1 — On-demand flow Shipped

Full end-to-end on-demand carousel delivery via Telegram.

Key decisions:
  • LLM switched from Gemini to Groq llama-3.3-70b-versatile — llama-3.1-8b-instant failed schema compliance
  • Schema enforcement: field names injected at call time via _SCHEMA_ENFORCEMENT block in writer.py
  • Caption: generated by LLM as top-level field (3–4 sentences, max 60 words, third person, no brand mention)
  • Caption format: {caption}\n\n{source_url} attached to first slide only (Telegram media group behaviour)
  • stat.number and signal.tag exempt from 3-char minimum validation

Phase 2 — Auto-daily flow Shipped (2A/2B shipped — 2C planned — 2D cancelled)

2A — ingestion.py ✅ Shipped 2026-06-11

  • Fetch: Serper (all topics) + RSS (8 feeds) + NewsAPI (top 3 topics)
  • Score: recency_weight × source_tier × signal_strength
  • Output: ranked top-10 candidate list → Telegram + store/candidates_{ts}.json
Key decisions:
  • Recency: exponential decay 0.5 ** (hours / 24), hard cutoff at 48h
  • Source tiers: A=8, B=6, C=4, D=2 from sources.yaml
  • Signal strength: 1.0 base + 0.1 per title keyword match, max 1.5
  • Source diversity cap: max 3 items per source before top-10 slice
  • Domain blocklist: stocktwits, reddit, twitter, x.com, bluesky, inc.com, fastcompany.com

2B — Auto-daily flow ✅ Shipped 2026-06-11

  • Scheduled run at 08:00 America/Toronto via APScheduler
  • Run ingestion, send candidate list to Telegram
  • Wait for approval reply: ALL / 1,3,5 / SKIP
  • 2-hour timeout on approval — skips run if no reply
  • Log run to store/run_log.json

Phase 3 — Images + Robustness Shipped

3A — image_fetcher.py

  • fetch_og_image(url) — GET OG image, validate ≥400×300px, convert RGB
  • fetch_flux_image(prompt) — FLUX.1-schnell via HuggingFace router, rotates 3 HF keys on 429
  • get_cover_image(og_image_url, image_query) — OG first, FLUX fallback, None if both fail

3B — Cover slide with real images

  • writer.generate() returns image_obj (PIL Image) alongside slides JSON
  • render_carousel() passes image_obj to both cover and context renderers
Key decisions:
  • OG image minimum size: 400×300px — smaller images rejected, FLUX used instead
  • Telegram media group hard cap: 10 slides max
  • Quote slide: 96px italic serif; Signal slide: headline 126px, implication 45px
  • Stat slide whitespace parked — body image scraping needed first (PARKING_LOT.md)
  • Curly quote bug: Claude Code inserted smart quotes — byte-replace all non-ASCII quotes

3E — Scraper robustness

  • Strategy chain: <p> tags → <article> → content divs → <main> → largest div → mobile UA → Jina Reader API
  • HTTP 403 responses skip all HTML strategies and go directly to Jina

3F — Scrape reliability (feedback.py + scorer.py)

  • Reliability formula: (attempts − failures) / attempts — multiplier applied after 3+ attempts
  • Domains below 30% reliability flagged to stderr
  • Needs 30+ days of live data before multiplier has meaningful scoring effect

3C — Regression baselines

9 baseline PNGs locked; 18 tests (9 dimension + 9 pixel-diff). Run: make regression.

3D — docs/design-system.html + docs/writer-contract.html

Full design token reference and LLM output contract; top nav updated across all 5 docs.

Phase 5 — Post-launch hardening Shipped

Ingestion reliability, image quality, scraper robustness, bot process management, ecosystem scaffolding.

5A — Logo feature

  • watcher/logo_fetcher.py — 28-company detection dict, Google S2 favicon API, local assets/logos/ fallback
  • renderer.py — logo paste above headline block on cover slide; 3× scaled, 85% opacity
  • writer.py — get_company_logo() called in generate(), returned as logo_obj
Known limitation: Google S2 returns favicons with white/grey backgrounds — local transparent PNGs in assets/logos/ needed for priority companies (Anthropic, OpenAI, Meta, etc.)

5B — Pexels image fallback

  • Image chain: OG → Pexels → FLUX → dark placeholder
  • PEXELS_API_KEY added to .env; portrait orientation filter
  • HF_API_KEY warning logged if all keys empty

5C — Source fallback on 403

  • _serper_alternative(headline) searches TC/Verge/Wired/Ars for alternative coverage
  • On scrape failure, retries writer.generate() on alt URL before sending user-facing error

5D — Body sanitization

  • scrape() sanitizes body before Groq: null bytes → control chars → UTF-8 round-trip → truncate 6000 chars
  • Fixes Groq 400 Bad Request on malformed scraped content

5E–5H — Ingestion + process fixes

  • 5E Serper dates: added week/month/month-name parser branches — "Jun 10, 2026" / "1 week ago" / "2 months ago" now correctly aged; stale stories no longer score as fresh
  • 5F Story dedup: semantic fingerprint (6 sorted key words, 4-word overlap threshold) — same story from multiple sources no longer fills top slots
  • 5G LaunchAgent: com.frameshift.bot.plist installed to ~/Library/LaunchAgents/; KeepAlive=true; logs to store/bot.log + store/bot.error.log
  • 5H arXiv URL: corrected to rss.arxiv.org — old arxiv.org/rss returned 302 that feedparser didn't follow

5I–5L — Ecosystem + docs

  • 5I DECISIONS.md: 13 ADRs extracted and documented — why, not just what
  • 5J RUNLOG.md: operational history from day one — first run, failures, fixes
  • 5K ~/shared/: ecosystem context bus — master-context.md, CIA/CCA context files, ig_aitech_posts.md performance log
  • 5L ~/oneninefour/: agent-patterns.md (5 patterns), ecosystem PARKING_LOT.md (8 entries)
Key decisions:
  • Serper dates all fell back to now() — "Jun 10, 2026" format not handled. Fixed with month-name parser branches and week/month relative handlers.
  • Story dedup was domain+title prefix only — same story from 3 sources filled top 3 slots. Fixed with semantic fingerprint; threshold=4 sufficient for same-day stories with shared vocabulary.
  • arXiv URL 302-redirected; feedparser never follows redirects — silent 0-item returns every run. Fixed by switching to rss.arxiv.org (returns 200 directly). arXiv 0 items on weekends is expected — no new submissions M–F deadline.
  • Bot was not persisting — manual terminal process dies on sleep/close. LaunchAgent with KeepAlive=true is the correct fix. Code changes now require launchctl unload/load cycle. See ADR-013.

Architecture Principles

No database
Flat JSON files only. Stories live as JSON per run, deleted after delivery.
Clean seams
Each module has one input type, one output type. Testable in isolation.
Ground or abstain
If source doesn't support a claim, omit the slide. Wired into prompt, not retrofitted.
Instruction-driven structure
Slide selection is a prompt, not code. Change behaviour by editing the prompt.
Fallback everywhere
Groq model fallback, font fallback, image fallback (OG → Pexels → FLUX → placeholder). Pipeline never dies silently.
Documentation ships with each phase, not after
tests/ and docs/ are built alongside features, never deferred.
