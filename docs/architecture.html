Frameshift v2 — Architecture

Last updated: 2026-06-11  ·  @ai.xr.frameshift

Overview

Frameshift v2 is a lean, two-mode carousel generation system running on a MacBook. It converts any article URL into a branded Instagram carousel of 4–8 slides, rendered at 1080×1350px, and delivers them to Telegram. No vector store, no dashboard, no ChromaDB — flat files and clean module seams.

Brand: The Frameshift  /  @ai.xr.frameshift
Built by: Snow Abad

Two modes

ModeTriggerTurnaroundStatus
On-demand Send /carousel <url> to Telegram bot ~5 min Shipped — Phase 1
Auto-daily Scheduled run → approval → batch delivery 3–5 carousels/day Shipped — Phase 2

Data Flow

On-demand flow

  1. User sends /carousel <url> command to Telegram bot
  2. bot.py receives command, validates URL, replies "Generating carousel..."
  3. writer.scrape(url) — 7-strategy fallback chain (p tags → article → divs → main → largest div → mobile UA → Jina Reader API); extracts title, body (max 8 000 chars), og:image URL
  4. writer.write(article) — builds prompt, calls Groq llama-3.3-70b-versatile (falls back to llama-3.1-70b-versatile on 429), validates JSON schema
  5. image_fetcher.get_cover_image() — fetches OG image (≥400×300px) or generates via FLUX, returns PIL Image or None
  6. renderer.render_carousel(slides, image_obj) — renders each slide to 1080×1350 PNG
  7. delivery.send(png_paths, data, url) — posts media group to Telegram, logs to store/delivered.json
  8. feedback.record_scrape_attempt(url, success) — updates domain reliability

Auto-daily flow

  1. APScheduler fires at configured time (default 8 am Toronto)
  2. ingestion.py — queries Serper API + NewsAPI + RSS feeds, scores by recency × source tier, deduplicates
  3. Ranked candidate list sent to Telegram for approval
  4. User replies ALL, story numbers (e.g. 1,3), or SKIP — 2-hour window
  5. For each approved story: writer.generate(url) → slides + caption + image
  6. renderer.render_carousel(slides) → PNG files in output/
  7. delivery.send() → Telegram media group for each carousel
  8. Run logged to store/run_log.json

Self-Learning System

The auto-daily pipeline includes a reinforcement learning feedback loop. Operator approval and rejection signals update source and topic scoring weights, improving candidate selection over time. Full explanation of the Contextual Bandit approach and weight update algorithm: Project Overview — Self-Learning.

Architecture Principles

No database
Flat JSON files only. Stories live as JSON per run, deleted after delivery.
Clean seams
Each module has one input type, one output type. Testable in isolation.
Ground or abstain
If source doesn't support a claim, omit the slide. Wired into prompt, not retrofitted.
Instruction-driven structure
Slide selection is a prompt, not code. Change behaviour by editing the prompt.
Fallback everywhere
Groq model fallback (llama-3.1-70b-versatile on 429), font fallback (DejaVu if TTFs absent), image fallback (OG → FLUX → dark placeholder). Pipeline never dies silently.
Documentation ships with each phase, not after
tests/ and docs/ are built alongside features, never deferred.
