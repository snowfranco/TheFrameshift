Frameshift v2 — Runbook

Last updated: 2026-06-11  ·  @ai.xr.frameshift  ·  watcher/bot.py is the entry point

Starting the System

  1. Ensure .env is populated — all 8 keys must be set: GROQ_API_KEY, TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, HF_API_KEY, HF_API_KEY_2, HF_API_KEY_3, SERPER_API_KEY, NEWS_API_KEY. HF keys 2 and 3 are optional (rotation); all others are required.
  2. cd ~/frameshift_v2
  3. python3 tests/smoke.py — run the full smoke test first. All 9 checks must pass before starting the bot.
  4. python3 watcher/bot.py
  5. Confirm startup log — you should see all three lines:
    [bot] starting — on-demand: /carousel <url> | daily: 8:00 AM Toronto [bot] scheduler armed — next run: 2026-06-12 08:00:00-04:00 [bot] polling... (Ctrl+C to stop)
  6. Scheduler is now armed. The daily job fires at 8:00 AM Toronto (America/Toronto). Next run time is shown in the startup log.

Common startup errors

  • telegram.error.Conflict: Terminated by other getUpdates request (409) Another bot instance is already polling. Fix: pkill -f bot.py, wait 5 seconds, restart.
  • [bot] TELEGRAM_BOT_TOKEN not set — check .env The bot exits immediately. Check ~/frameshift_v2/.env — the key must be on its own line with no extra spaces.
  • GROQ_API_KEY is not set — check ~/frameshift_v2/.env Raised when the first /carousel is sent. Set GROQ_API_KEY in .env and restart the bot.

On-Demand Carousel (/carousel)

Send /carousel <url> in the Telegram chat. Any valid http:// or https:// URL is accepted.

Expected flow

  1. Bot replies "Generating carousel..." immediately
  2. Scrape → Groq write → image fetch → Pillow render (~2–3 min)
  3. Slides delivered as Telegram media group (4–8 PNGs)
  4. Caption + source URL attached to the first slide
  5. Run logged to store/run_log.json

Failure messages

  • ⚠️ Couldn't fetch that article. Site is paywalled, bot-blocked, or JavaScript-rendered. All 7 scrape strategies failed including Jina Reader API. Try: archive.ph/<url> or 12ft.io/<url>.
  • Generation hiccup — retrying... then Generation failed: <reason> Groq returned invalid JSON or a schema violation. Bot retried once automatically. Check the reason — "Invalid carousel JSON" usually means a field name mismatch. Running again often clears it.
  • Generation failed: caption must be at least 80 characters Groq returned a very thin summary — article content was likely too short. Retry with a longer article, or check that the scrape extracted real content.

Auto-Daily Flow

Schedule: 8:00 AM Toronto (America/Toronto)
Manual trigger: Send /run in Telegram (owner chat only)

Expected flow

  1. Bot runs ingestion — fetches Serper, RSS feeds, and NewsAPI; scores by recency × source tier × signal strength
  2. Sends candidate list to Telegram — up to 10 stories with scores and summaries
  3. You reply within 2 hours with one of:
    • ALL — generate all candidates
    • 1,3,5 (or 1 3 5) — generate stories 1, 3, and 5 only
    • SKIP — skip this run entirely
  4. Bot generates + delivers approved carousels, one by one
  5. Run complete — bot replies "batch complete — N carousel(s) delivered"
  6. Each delivery logged to store/run_log.json
If no reply within 2 hours, the approval window expires and the run is skipped. The pending candidates file (store/pending_candidates.json) is cleared automatically.

Test Commands

CommandWhat it testsWhen to run
python3 tests/smoke.py Full pipeline end-to-end (9 checks): scrape → write → caption → slide structure → image → render → dimensions → delivery Before any deploy or bot restart
make regression Pixel-diff against 9 approved PNG baselines — 18 checks total (dimensions + exact match) Before any renderer or design_constants change
make test All unit tests + regression suite After any code change
python3 tests/smoke.py <url> Smoke test with a specific article URL Debugging a source that's failing in production

Test suite coverage

Smoke test (tests/smoke.py)
Live network calls, no mocks. Covers: scrape, Groq write, schema validation, image fetch, Pillow render, PNG dimensions, Telegram delivery.
Regression tests (tests/regression/)
9 slide types × 2 checks = 18 tests. Dimension check (1080×1350) and zero-diff pixel comparison against locked baselines. Run with make regression.
Unit tests (tests/unit/)
Writer schema validation (rejects bad JSON, accepts valid) and renderer output (each slide type renders, correct size, spot colour checks).
Not covered by automated tests
Ingestion scoring logic, FLUX image generation fallback, Groq model fallback (429 retry), approval parsing edge cases. These are exercised by running the daily flow manually.

Updating Regression Baselines

Run this after any intentional change to watcher/renderer.py or watcher/design_constants.py. The regression suite will fail until baselines are refreshed — that failure is the safety net, not a bug.

  1. Make the renderer or design constants change
  2. Visually verify the output looks correct — render the test carousel and open the PNGs in Preview: python3 -c "import sys; sys.path.insert(0,'watcher'); from renderer import render_carousel, TEST_CAROUSEL; render_carousel(TEST_CAROUSEL, prefix='preview')"
  3. make update-baselines — regenerates all 9 baseline PNGs
  4. make regression — all 18 tests must pass
  5. Commit code changes and updated baselines together in the same commit
  6. Update docs/design-system.html if any pixel values, colours, or font sizes changed

Daily Context Update

Run after any session where something ships. Paste this prompt into Claude Code. It marks completed phases, updates context.md, and regenerates docs/roadmap.html. 

Read ~/frameshift_v2/ROADMAP.md and ~/frameshift_v2/docs/context.md

Run the wrap-up sequence:
1. Mark anything shipped this session as complete in ROADMAP.md
2. Update context.md — current phase, next task, known issues. Max 60 lines.
3. Regenerate docs/roadmap.html from the updated ROADMAP.md
4. Report what was updated and what the next task is

Read before write. Diff before save on every file.

The full daily update protocol is documented in Overview — Daily Context Update.

Adding a New Slide Type

Run make regression before starting — lock the current baseline so you have a clean reference point.
  1. Design the slide — build a React mockup, get visual approval before writing any code
  2. Add to watcher/design_constants.py:
    • Append to SLIDE_TYPES list
    • Add schema entry to WRITER_SCHEMA with all fields + word limits
    • Add palette rule to WRITER_PROMPT with "use when…" instruction
  3. Add renderer function to watcher/renderer.py:
    • Write render_{type}(data) → PIL.Image following existing pattern
    • Add entry to RENDERERS dict — nothing else changes
  4. Add the new type's fields to _SCHEMA_ENFORCEMENT in watcher/writer.py
  5. Add a fixture entry to REGRESSION_CAROUSEL in tests/regression/test_renderer_regression.py and add the type to SLIDE_TYPES in that file
  6. make update-baselines — generates baseline PNG for the new type
  7. make regression — all tests must pass (including the new type's dimension + pixel-diff)
  8. Update docs/design-system.html (new slide-type subsection) and docs/writer-contract.html (new schema subsection)

Common Failures and Fixes

SymptomCauseFix
409 Conflict on bot startup Another bot instance is already polling pkill -f bot.py, wait 5 s, restart
category validation error — "too short" Groq used a 1-char category (e.g. "AI" before the fix) Already fixed — 2-char minimum in _FIELD_MIN. If it resurfaces, check writer.py:_FIELD_MIN.
Caption too short / generation fails Groq returned a thin summary on sparse article content Retry the command. If persistent, check that scrape extracted real body text (>200 chars).
Stat slide has excess whitespace Short stat content on a tall canvas Known issue — parked in PARKING_LOT.md. Requires body image scraping to resolve.
Curly quotes / syntax error in renderer.py Claude Code inserted Unicode smart quotes into Python string literals Run a byte-level check after any renderer edits: grep -P '[‘’“”]' watcher/renderer.py. Replace with ASCII equivalents.
Regression tests fail Renderer was changed without updating baselines make update-baselines then make regression
Serper or NewsAPI returns 0 items API key quota exceeded or key expired Check keys in .env. Verify against Serper and NewsAPI dashboards.
OG image not appearing on cover slide OG image was <400×300px, or the fetch failed FLUX fallback triggers automatically. If FLUX also fails, renderer uses dark placeholder — not an error.
Bot does not respond to commands Bot process has stopped or was never started cd ~/frameshift_v2 && python3 watcher/bot.py
