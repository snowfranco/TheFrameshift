Frameshift v2

A self-learning multi-agent carousel system
Last updated: 2026-06-11  ·  @ai.xr.frameshift

What Is This

Frameshift v2 is a multi-agent AI system that monitors signals across AI, spatial computing, AR, and emerging technology, then automatically produces branded Instagram carousels for The Frameshift (@ai.xr.frameshift). It operates in two modes: on-demand (send a URL, get a carousel) and auto-daily (scheduled ingestion, human approval, batch generation). The system is self-learning — approval and rejection signals from the human operator continuously improve source scoring and story selection over time.

Multi-Agent Architecture

The system is composed of discrete agents, each with a single responsibility and a clean input/output contract. Agents communicate through typed data structures — no shared state, no side effects across module boundaries.

Self-Learning: Reinforcement Learning from Human Feedback

The system uses a feedback loop pattern known as a Contextual Bandit — a lightweight reinforcement learning approach where the system presents options (candidate stories), observes which ones the operator approves or rejects, and updates its scoring weights to improve future selections. No neural network required at this scale.

Every carousel generation attempt is logged to store/feedback_log.json with domain, URL, timestamp, and success/failure. feedback.py updates the domain_reliability score in scoring_weights.json: successful scrapes push reliability toward 1.0, failures toward 0.0. After 3+ attempts on a domain, scorer.py applies this as a multiplier to that domain's candidate scores — consistently unscrapeable domains are automatically deprioritised. A domain below 30% reliability after 3+ attempts triggers a blocklist warning to stderr.

The system starts with source tiers and recency decay as priors and improves with use. After 30+ days of data, the scoring reflects which domains are reliably accessible — not just editorially high-tier but actually scrapeable. Note: editorial approval/rejection signals (which stories the operator selects) are not yet fed back into scoring; that is Phase 2C. The current feedback loop improves source accessibility, not editorial preference.

Context Management

Frameshift v2 implements a file-based, agent-first context management system. Rather than using a vector store or framework like LangChain, the system maintains context through structured documents that both humans and AI agents can read, navigate, and update directly.

LLM agents navigate by file path and section ID more reliably than by semantic retrieval at this project scope. Explicit structure beats implicit retrieval when the knowledge base is small and stable. Every doc has a machine-readable id on every section, data-status attributes on every status item, and a <meta name="agent-context"> tag describing what the file covers.

The system is organized around four pillars:

Just-in-Time Retrieval
Claude Code reads specific files before acting. Prompts explicitly name which files to read. No file is loaded unless the task requires it.
State Management
ROADMAP.md is the single source of truth for project state. store/ holds flat JSON run logs. scoring_weights.json persists learned weights across sessions. No database.
Compression & Summarization
docs/context.md is the context bootstrap — 39 lines that orient any agent completely in one read. Replaces walls-of-text handoff summaries.
Session Continuity
The project lives in a dedicated Claude Project with context.md pinned as the project instruction. Every new session starts from the same known state.

Scoring Formula

score = recency_weight × source_tier × signal_strength
FactorDescriptionRangeNotes
recency_weight Exponential decay from publish time 0.0–1.0 24hr = 1.0, 48hr ≈ 0.6, 7d ≈ 0.1
source_tier Editorial quality of source 2–8 A=8, B=6, C=4, D=2; adjusted continuously by feedback.py
signal_strength Relevance to topic set 1.0–1.5 Keyword match score against sources.yaml topics

source_tier values are initial priors. feedback.py adjusts them continuously based on operator approval signal — a source with consistent approvals drifts upward; zero approvals in 30 days triggers automatic demotion.
